minecraft sounds are weird.

now, in minecraft, if you're trying to make a resource pack, it's relatively simple to get a base with most textures. you just need to open up the version's JAR file (by default in windows, %appdata%/.minecraft/versions/ )in compression software like WinRAR, and enter the assets folder.
however, if you want to have sounds as well, it's a bit more complicated.

to get sounds, you first need to understand the structure of the assets folder. there are two folders inside assets we need; indexes and objects. inside indexes is a .json file listing every single sound in that version, along with a hash of it, and its size in bytes (used during startup to verify files) to find a file, first search the newest version's.json for the sound you want; i'll be assuming we want to play the music Left to Bloom. when searching for a given file, you should always search without capitalization, and replacing any spaces with underscores; instead of Left To Bloom, we're gonna look for left_to_bloom. now, according to the index we're using, the sound minecraft/sounds/music/game/left_to_bloom.ogg has the hash aa1d3aace1c481ac32d5827fba287294b6bc99fb. now with the hash, we enter the objects folder. inside objects , there are 256 folders, from 00 to ff. the first two letters of our hash are the folder we need to enter; in this case they're aa, so we enter the aa folder. inside aa, there are many files, each with a different hash as a different file. however, there's one with our hash as its name, and that's the one we're after. rename it to left_to_bloom.ogg, and you've got Left To Bloom!

now, you might be thinking, "hey, this whole process takes a while, is it automated yet?"
